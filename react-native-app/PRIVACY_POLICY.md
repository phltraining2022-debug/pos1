# Privacy Policy — Kara POS

**Last Updated:** May 16, 2026

## Overview
Kara POS ("App") is a point-of-sale system for restaurants and food service businesses. This Privacy Policy explains how we collect, use, and protect your information.

## Information We Collect

### User Authentication
- **Login Credentials:** Username and password (transmitted securely over HTTPS/TLS 1.2)
- **User ID & Role:** Stored locally on device to maintain user session
- **Authentication Token:** Used to verify identity with backend services

### Device Information
- **Device Token (APNs):** For push notification delivery
- **Device Model & OS Version:** For diagnostic and compatibility purposes
- **Device Name:** Associated with device registration for push notifications

### Order & Transaction Data
- **Table Information:** Table names, numbers, status
- **Order Details:** Items ordered, quantities, prices, timestamps
- **Payment Information:** Payment method used (cash, card, digital wallet)
- **Staff Information:** Assigned cashier, waiter, manager

### Photos & Media (if used)
- **Product Images:** Images uploaded for inventory management
- **Receipt Images:** If user exports receipt photos
- **Camera Access:** Only when user explicitly grants permission for QR code scanning

## How We Use Your Information

1. **Authentication & Session Management**
   - Verify user identity
   - Maintain secure login sessions

2. **Order Processing & Payment**
   - Record transactions for business reporting
   - Generate receipts and reports

3. **Push Notifications**
   - Notify staff of new orders, table changes, system updates

4. **Diagnostics & Improvements**
   - Identify bugs and performance issues
   - Improve app stability and user experience

## Data Security

- **Encryption in Transit:** All data is encrypted using HTTPS/TLS 1.2 or higher
- **Local Storage:** User authentication tokens are stored securely in device keychain
- **No Sensitive Data Cached:** Payment card numbers are never stored locally
- **Server Security:** Backend uses industry-standard security practices

## Data Retention

- **Session Data:** Cleared upon logout
- **Transaction Records:** Retained on server as per restaurant business requirements (typically 7–30 days)
- **Device Tokens:** Retained until app is uninstalled or notification permission revoked

## Permissions Requested

### Camera
- **Purpose:** Scan QR codes for table/item selection
- **User Control:** Permission requested at first use; can be revoked in Settings

### Photo Library
- **Purpose:** Upload product images to inventory
- **User Control:** Permission requested at first use; can be revoked in Settings

### Face ID / Biometric
- **Purpose:** Optional quick login authentication
- **User Control:** User enables in app settings; can be disabled anytime

### Notifications
- **Purpose:** Receive order and system updates
- **User Control:** Permission requested at login; can be revoked in Settings

## Third-Party Services

This app may integrate with:
- **Backend API Server:** For data synchronization (kara.live1.vn, kara.app.live1.vn)
- **Cloud Services:** For push notifications (Apple Push Notification Service)
- **Analytics:** May use diagnostic tools to monitor app performance (no personal data sold)

## Your Rights

- **Access:** You can request your data stored on our servers
- **Deletion:** Contact support to delete your account and associated data
- **Opt-Out:** You can disable notifications and camera access anytime in Settings

## Children's Privacy

This app is designed for business use and is not intended for children under 13. We do not knowingly collect information from children.
